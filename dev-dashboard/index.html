• backend services

    • S3ULMU-HL7-Interface server
      Service that receives HL7 messages from the KUM comm server eGate. Is intended to run 24/7/365.

    • S3ULMU-Recruiter
      Service that checks the S3ULMU-HL7-Interface's database on a daily basis for patients that fit the trials inclusion/exclusion criteria defined at S3ULMU-Studienmonitor. Writes 'hits' to the S3ULMU-Webinterface and informs physicians and trial personell by mail.

    • S3ULMU-Feasibility Check
      Service that checks the S3ULMU-HL7-Interface's database on a daily basis for patients that fit the feasibility requests defined at S3ULMU-Webrequest. Writes 'hits' to the S3ULMU-Webrequest and informs scientists and administrators by mail.

    • S3ULMU-Eraser
      Service that checks the S3ULMU-HL7-Interface's database on a daily basis for patients that have been discharged 48 or more hours ago. If any datasets are found, they will be deleted for privacy reasons.

  • frontend ui's

    • S3ULMU-Studienmonitor (Mockup)
      Dummy/Placeholder application to demonstrate registration of KUM trials and their (structured) inclusion/exclusion criteria. Features will probably be migrated to CCSB developed by Anja Fischer

    • S3ULMU-HL7-Interface database
      Database of the S3ULMU-HL7-Interface server.

    • S3ULMU-Webinterface
      Interface for physicians to view and consent patients that fit trial inclusion/exclusion criteria of the S3ULMU-Studienmonitor.
      WARNING: requires startup of S3ULMU-Studienmonitor and S3ULMU-Consent Manager

    • S3ULMU-Consent Manager
      (possibly) Internet-exposed self-service application for patients managing their informed consents (or physicians viewing the consents of their patients which have been automatically transfered from the S3ULMU-Webinterface).
      WARNING: requires startup of S3ULMU-Studienmonitor

    • S3ULMU-Webrequest
      Internet-exposed application which gives interested scientists the ability to check for cohort sizes of certain criteria (similar to S3ULMU-Studienmonitor but with anonymized/aggregated results). Might fit for m4 TSC or KUM CSC processes.
      WARNING: requires startup of S3ULMU-Studienmonitor

    • S3ULMU-Studiendatenbank (Mockup)
      dummy/placeholder application to demonstrate data communication from S3ULMU-Webinterface to an arbitrary CDMS/CDBMS
